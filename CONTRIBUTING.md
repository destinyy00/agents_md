# Contributing Guide

Thank you for considering improvements to Blueprint Registry.

## Workflow

1. Fork the repository.
2. Create a feature branch with a descriptive name.
3. Make focused changes with clear commit messages.
4. Run:

```bash
npm install
npm run build
```

5. Open a pull request with:
- Problem statement
- What changed
- Why this approach was chosen
- Screenshots for UI changes

## Standards

- Keep the experience static-first and fast.
- Do not add external AI APIs.
- Preserve the guided flow (category first, result last).
- Maintain anti-generic UI direction from `design.MD`.
- Ensure mobile responsiveness and keyboard accessibility.

## Suggested Areas

- Additional blueprint templates
- Better category/keyword mapping
- Form and validation improvements
- Accessibility upgrades
- Build and deployment automation
